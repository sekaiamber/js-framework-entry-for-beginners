库 vs 框架

为解决某一具体问题而拼凑的一大堆函数和类集合。

工具，被调用者

velocity.js
fairy.js
...
某种已经具有一定功能的项目半成品或原型。

框架，规则制定者

jQuery
Angular
...

大不同

然而这个界限正变得越来越模糊！

库越来越参与规则制定：

// velocity.js
$('#foo').velocity({
    width : '100px',
    opacity : .5
});
// fairy.js
fairy('init', {
    transitionDuration : 1000
})('start');
框架越来越趋于工具化：

// jQuery
jQuery.noConflict();  
(function($) {
    $(function() {
        // some code using `$`
    });
})(jQuery);
Javascript框架分类（架构、核心思想）
  1. 命名空间导向，Java风格，`new`，如YUI、EXT 
    
Ext.MessageBox.alert("Example", "using namespace.");
  2. 类工厂导向，工厂模式，如Prototype、mootools 
    
var Animal = new Class({
    initialize: function(name){
        this.name = name;
    }
});
var Cat = new Class({
    Extends: Animal,
    talk: function(){
        return 'Meow!';
    }
});
var pet = new Cat('Tom');
  1. 选择器导向，如jQuery 
    
$('#foo').addClass('bar');
$.ajax({
    url : 'you-path'
});
$('#foo').plugin();
  2. 加载器导向，模块化，如Dojo、YUI 
    
dojo.require("dojo.parser");
dojo.require("dijit.layout.ContentPane");
dojo.require("dijit.layout.TabContainer");
  3. 架构导向，MV*，如Angular、backbone.js
Javascript框架都做了点什么？
  • DOM操作
  • 语言扩展
  • 浏览器功能修正
  • Ajax支持
  • 选择器
  • 浏览器嗅探(*)和特性侦测
  • 事件代理
  • 数据缓存
  • 动画引擎
  • 某种扩展或增强方式
  • 异步执行解决方案
  • 类工厂(*)
  • 兼容性
  • ...
